# Utils (`utils/`)

Build scripts, shell utilities, and development tooling.

## Files

| File | Purpose |
|------|---------|
| `launch.sh` | Main launch script — builds the C++ app (CMake + Ninja), builds the React UI (`npm run build`), and launches `SongbirdPlayer.app`. Handles both release and debug builds. |
| `local.sh` | Local development helper — shortcut for building and running locally. |
| `setup.sh` | First-time setup — installs Arduino CLI, board cores, and libraries (legacy, from FeatherS2 era). |
| `flash.sh` | Flashes files to FeatherS2 board via USB (legacy hardware support). |
| `files.sh` | File management utility for the FeatherS2 FATFS partition (legacy). |
| `build_bindings.json` | VSCode keyboard shortcut bindings for build tasks (Cmd+R = build, Cmd+B = debug). |
| `include_path.json` | C++ include paths for IDE IntelliSense. |
| `mkfatfs` | Binary tool for creating FAT filesystem images (FeatherS2 flashing). |
| `partition/` | ESP32-S2 partition table definitions (legacy hardware). |

## Build Workflow

The primary build workflow for the desktop app:

```bash
# Full build (C++ + React UI)
./utils/launch.sh

# Or manually:
cmake -B build -G Ninja
cmake --build build
cd react_ui && npm run build
open build/SongbirdPlayer_artefacts/Debug/SongbirdPlayer.app
```

> **Note:** The `setup.sh`, `flash.sh`, `files.sh`, and related Arduino/ESP32 tools are from the original hardware prototype era. The current Songbird is a desktop-only macOS app.
